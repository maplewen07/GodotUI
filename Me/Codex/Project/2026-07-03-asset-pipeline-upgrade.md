# 2026-07-03 Asset Pipeline Upgrade

- Task: complete a practical Godot plugin asset pipeline for manifest UI packages.
- Manifest: `assets.json` now supports `texture`, `font`, and `theme` assets under `ui/<system>/assets/`, with hash, size, optional import data, default font settings, and theme type variations.
- Generator: validates asset containment, traversal, existence, kind/extension compatibility, hash mismatch, missing theme font assets, and missing texture asset references. It emits stable `res://` asset refs and generates derived theme resources such as `generated/ui/phone/assets/PhoneTheme.tres`.
- Runtime: added `ManifestAssetRef` and `ManifestAssetCatalog`; texture conversion resolves asset ids through the generated catalog, and generated widgets apply the generated theme on ready.
- Editor plugin: added `Import Assets`, which refreshes Godot's editor filesystem and reimports the selected package asset files without running a whole-project headless import subprocess.
- Sample: `ui/phone` now includes texture, font, and theme sample assets; `PhoneWidget.g.cs` emits the generated catalog and theme path.
- Tests: `SelfCheck` verifies texture loading, generated theme loading, default font/default font size, and `themeClass`. Negative fixtures cover asset location, hash mismatch, kind/extension mismatch, missing theme font, and missing texture asset id.
- Verification:
  - `dotnet build tools\ManifestUiGen\ManifestUiGen.csproj`
  - `dotnet run --no-build --project tools\ManifestUiGen -- validate ui\phone\package.json --write-report`
  - `dotnet run --no-build --project tools\ManifestUiGen -- migrate --check ui\phone\package.json`
  - `dotnet run --no-build --project tools\ManifestUiGen -- generate ui\phone\package.json`
  - negative fixture loop for asset and binding failures
  - `dotnet build GodotUI.csproj`
  - `godot --headless --path . --import`
  - `godot --headless --path . --scene res://tests/SelfCheck.tscn`
  - `godot --headless --path . --scene res://demo/Main.tscn --quit-after 5`
  - `godot --headless --build-solutions --quit` exited 0, with a GodotTools hot-reload timer log emitted by the editor tooling.
- Skipped: no copying source assets into generated output, atlas generation, compression policy, platform overrides, complex theme token conversion, or automatic import on every generate.
