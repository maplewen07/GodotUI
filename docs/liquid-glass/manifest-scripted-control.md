# Manifest UI Scripted-Control Integration

Manifest UI and Liquid Glass remain independent addons. Integration is opt-in
at authoring time through Manifest UI's generic scripted-control and resource
extension points; neither addon references the other's assemblies.

After installing both addons, a non-root layout node can attach a Liquid Glass
control script:

```json
{
  "id": "glass_panel",
  "type": "Control",
  "name": "GlassPanel",
  "scriptPath": "res://addons/liquid_glass/runtime/LiquidGlassPanel.cs",
  "properties": {
    "offset_left": 48,
    "offset_top": 48,
    "offset_right": 432,
    "offset_bottom": 264,
    "Preset": 4,
    "Style": { "assetRef": "frosted_superellipse_style" },
    "accessibility_name": "Inventory summary"
  },
  "children": []
}
```

Declare the style as a generic resource asset:

```json
{
  "assets": [
    {
      "id": "frosted_superellipse_style",
      "relativePath": "assets/frosted_superellipse_style_resource.tres",
      "kind": "resource"
    }
  ]
}
```

`type` remains `Control`; Godot Engine Verify inspects the attached script's
exported properties and signals. C# exported layout properties use the engine
names generated by Godot (`Preset`, `Style`, `Text`, and `Value`), while
GDScript properties retain their authored snake_case names. Canonical Manifest behavior is declared in
`bindings.json`, exactly as it is for built-in controls:

```json
{
  "inputs": [
    {
      "id": "intensityInput",
      "target": "IntensitySlider",
      "property": "value",
      "source": "viewmodel:intensity"
    }
  ],
  "controls": [
    {
      "id": "ActionButton",
      "target": "ActionButton",
      "events": [
        {
          "event": "pressed",
          "handler": "apply_glass",
          "channel": "liquid_glass.apply"
        }
      ]
    }
  ]
}
```

The generated Widget maps canonical `pressed` and `value_changed` declarations
to the C# engine signals `Pressed` and `ValueChanged`; the Controller
only reads and writes the Store. Removing Liquid Glass makes this package fail
Engine Verify or scene loading, but does not change Manifest UI itself.

The repository integration package is `ui/liquid_glass/package.json`. The
standalone `examples/LiquidGlass` project intentionally remains independent of
Manifest UI and is still the source for the Liquid Glass release example.
